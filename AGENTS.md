# Project Rules

## Git Workflow
- After completing each meaningful, self-contained task (e.g., implementing a feature, fixing a bug, finishing a refactor), create a git commit with a clear, descriptive commit message.
- Do not leave changes uncommitted.
- Use conventional commit prefixes: `feat:` for new features, `fix:` for bug fixes, `refactor:` for refactoring, `chore:` for maintenance tasks, `docs:` for documentation, `test:` for tests, `style:` for formatting changes.

## Code Style
- Do NOT write comments in the code. No inline comments, no block comments, no docstrings, no TODO comments. The code should be self-explanatory through clear naming and structure.
